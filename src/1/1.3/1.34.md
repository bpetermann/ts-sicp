## Exercise 1.34

> Suppose we declare
>
> ```js
> function f(g) {
>   return g(2)
> }
> ```
>
> Then we have
>
> ```js
> f(square)
> ```
>
> 4
>
> Then we have
>
> ```js
> f((z) => z * (z + 1))
> ```
>
> 6
>
> What happens if we (perversely) ask the interpreter to evaluate the application f(f)? Explain.

## Solution

Will end in a TypeError: Uncaught TypeError: g is not a function, because f(f) => f(2) => 2(2)
